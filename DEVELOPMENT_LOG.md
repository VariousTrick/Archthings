# Development Log

## 2026-04-05

### 仓库整理原则

- 只提交我们自己写的 shell、脚本、主题和文档
- 不提交外部参考仓库
- 不让运行结果依赖 `HyDE`、`DMS`、`orbit`、`shorin-niri` 等参考项目目录

### Waybar

- 修复了 `waybar` 在 `niri` 中跟随工作区切换的问题
- 最终有效修法：
  - 去掉 `waybar` 配置里的 `mode: dock`
  - 把 `waybar` 置于 `top` layer
  - 为 `wallpaper` 增加 `place-within-backdrop true`
  - 在 `niri` 中把背景改为透明，从而让 overview/backdrop 行为正常

### 电池面板

- 用 `Quickshell` 做出了独立电池面板原型
- 后来又改成：
  - 常驻实例
  - 点击按钮时几乎即时显示
- 已实现：
  - 电量信息
  - 电量条
  - 电源模式切换
  - 80% 充电限制
  - 拖拽把手
  - 位置记忆
  - hover 反馈

### 音量面板

- 做出了独立 `Quickshell` 音量面板
- 已实现：
  - 扬声器 / 麦克风音量
  - 图标点击静音
  - 100% 上限滑条
  - 输入 / 输出设备切换
  - 二级设备列表
  - hover 高亮

### Wi‑Fi 面板

- 做出了独立 `Quickshell` Wi‑Fi 面板
- 当前实现：
  - Wi‑Fi 总开关
  - 已连接 / 可用网络分区
  - 可滚动网络列表
  - 加密网络行内密码输入
  - 隐藏网络手动连接
  - 刷新按钮
  - 密码输入的取消 / 收回路径
  - 密码明文显示切换

### 蓝牙面板

- 推倒重写了一版蓝牙面板基线，放弃前面那套混杂的配对状态机补丁
- 当前这一版故意只保留稳定的一层：
  - 蓝牙总开关
  - 扫描 / 刷新
  - `已连接 / 已配对 / 设备` 三个分组
  - 每行一个主动作：
    - `断开`
    - `连接`
    - `配对`
  - 已配对设备的 `忘记` 次级动作
  - 行尾按钮 hover 高亮
- 这一版的目标不是直接完成 PIN / 配对确认，而是先把：
  - 面板稳定弹出
  - 列表稳定刷新
  - 主按钮稳定点击
  这三个基础点立住
- 后续如果继续做蓝牙 PIN / 配对确认，会作为下一阶段单独实现，不再和列表面板纠缠在一起

### 依赖调整

- 增加了 Wi‑Fi 面板运行依赖：
  - `networkmanager`
  - `nmcli`
- 当前仓库里的 Wi‑Fi 面板入口已不再依赖 `orbit-wifi`
- `orbit` 目前仍可作为蓝牙 / 旧入口的参考或备用，但不再是 Wi‑Fi 面板的必需依赖

### Waybar 辅助脚本

- 形成了 `.codex-waybar` 目录，集中保存：

  - 早期原型脚本和主题文件

### 风格方向

- 以 `waybar` 当前深灰配色为基准
- 参考 Plasma 的结构与观感
- 不直接复制 Plasma / DMS / HyDE 的实现
- 统一走“参考结构 + 自己实现”的路线

### 系统清理经验

- `shorin-niri` 曾留下大量坏软链接，影响了：
  - 输入法
  - 默认打开方式
  - GTK 主题
  - 图标与通知
- 经验结论：
  - 以后尽量避免让核心配置直接依赖外部 dotfiles 仓库的软链接
  - 仓库里只保留本地可独立使用的真实文件
